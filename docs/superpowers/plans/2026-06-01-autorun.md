# Autorun Script Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Algo.exe와 MyCar(java)를 N번 자동 반복 실행하는 PowerShell 스크립트를 작성한다.

**Architecture:** `Simulator/autorun.ps1` 단일 파일. 파라미터로 반복 횟수·워밍업·쿨다운을 받아, 루프마다 Algo.exe 시작 → java 실행(완료 대기) → Algo.exe 종료 순서로 처리한다. 두 실행 파일이 다른 디렉터리에 있으므로 `$PSScriptRoot` 기준 상대경로로 각각 WorkingDirectory를 지정한다.

**Tech Stack:** PowerShell 5.1+ (Windows 내장), Java 8+ (기존 환경)

### Task 1: `autorun.ps1` 작성

**Files:**
- Create: `Simulator/autorun.ps1`

- [ ] **Step 1: 파일 생성**

`Simulator/autorun.ps1`을 아래 내용으로 작성한다.

```powershell
param(
    [int]$Runs    = 5,
    [int]$Warmup  = 3,
    [int]$Cooldown = 2
)

$AlgoExe   = "$PSScriptRoot\Algo.exe"
$BotJavaDir = "$PSScriptRoot\..\Bot_Java"

# 사전 확인
if (-not (Test-Path $AlgoExe)) {
    Write-Error "Algo.exe not found: $AlgoExe"
    exit 1
}
if (-not (Test-Path $BotJavaDir)) {
    Write-Error "Bot_Java directory not found: $BotJavaDir"
    exit 1
}

for ($i = 1; $i -le $Runs; $i++) {
    Write-Host "=== Run $i/$Runs ===" -ForegroundColor Cyan

    # 1. 시뮬레이터 시작
    $algo = Start-Process -FilePath $AlgoExe `
                          -ArgumentList "-ResX=640 -ResY=480 -windowed" `
                          -PassThru
    Start-Sleep -Seconds $Warmup

    # 2. MyCar 실행 (종료될 때까지 대기)
    $java = Start-Process -FilePath "java" `
                          -ArgumentList "-cp .;DrivingInterface -Djava.library.path=DrivingInterface MyCar" `
                          -WorkingDirectory $BotJavaDir `
                          -NoNewWindow `
                          -Wait `
                          -PassThru
    Write-Host "Exit code: $($java.ExitCode)"

    # 3. 시뮬레이터 종료
    Stop-Process -Name "Algo" -Force -ErrorAction SilentlyContinue
    Write-Host "Algo stopped. Waiting ${Cooldown}s..."
    Start-Sleep -Seconds $Cooldown
}

Write-Host "All $Runs runs complete." -ForegroundColor Green
```

- [ ] **Step 2: 경로 확인**

Windows에서 `Simulator/` 디렉터리 안에 파일이 있는지 확인한다.

```
dir Simulator\
```

예상 출력에 `autorun.ps1`, `Algo.exe`, `run.bat`가 포함되어야 한다.

- [ ] **Step 3: 파라미터 없이 dry-run (경로 검증만)**

PowerShell 터미널에서 `-Runs 0`으로 루프 없이 경로 검증만 실행한다.

```powershell
cd Simulator
PowerShell -ExecutionPolicy Bypass -File autorun.ps1 -Runs 0
```

예상 출력:
```
All 0 runs complete.
```

오류가 없으면 경로 설정이 올바른 것이다. `Algo.exe not found` 오류가 나오면 스크립트 내 `$AlgoExe` 경로를 점검한다.

- [ ] **Step 4: 1회 실행으로 동작 확인**

IntelliJ에서 MyCar를 먼저 빌드(컴파일)한 뒤 실행한다.

```powershell
PowerShell -ExecutionPolicy Bypass -File autorun.ps1 -Runs 1 -Warmup 3
```

예상 출력:
```
=== Run 1/1 ===
[MyCar] Start Bot! (JAVA)
...
[MyCar] End Bot! (JAVA), return_code = 0
Exit code: 0
Algo stopped. Waiting 2s...
All 1 runs complete.
```

MyCar 출력이 보이지 않으면 `-NoNewWindow` 제거 후 재시도한다(별도 창에서 출력될 수 있음).
